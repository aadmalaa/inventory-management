System Architecture

Factory Inventory Management System

A full-stack demo application: Vue 3 single-page app talking to a FastAPI backend over REST, with all data held in memory from JSON seed files — no database, no authentication. Built as a Claude Code workshop playground.

System Overview

Three tiers: a browser-based SPA, a Python API server, and a JSON-file data layer loaded once at startup.

Data Flow

The core loop: user picks filters, every view re-fetches, the backend filters in plain Python, and Vue computed properties derive everything shown on screen.

1
FilterBar
User sets Time Period, Warehouse, Category, Order Status
2
useFilters
Shared refs update; views watch and re-fetch
3
api.js
axios GET with query params (skips values of “all”)
4
FastAPI
apply_filters + filter_by_month over in-memory lists
5
Pydantic
Response validated against response_model
6
Vue refs
Raw data lands in refs (allOrders, inventoryItems…)
7
Computed
KPIs, tables and SVG charts derive from computed properties

Known Gaps

The frontend API client calls endpoints the backend doesn't implement. These will return 404 until routes are added to server/main.py:
  • /api/tasks (GET/POST/PATCH/DELETE) — used by TasksModal.vue
  • POST /api/purchase-orders and GET /api/purchase-orders/{backlogItemId} — the Pydantic models (PurchaseOrder, CreatePurchaseOrderRequest) exist, but no routes are wired up
Other demo-only constraints: no authentication, wildcard CORS, no pagination, and nothing persists across a server restart.
